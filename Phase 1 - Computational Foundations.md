# Phase 1 — Computational Foundations
## AI Engineering Product School Curriculum

Phase 1 establishes the **fundamental layer of engineering competence** required for all advanced software and AI engineering work.  

The objective is to ensure engineers **fully understand how computing systems function internally** before progressing to higher layers such as software architecture, data engineering, and artificial intelligence systems.

This phase focuses on **computational literacy, systems thinking, and command of core computing environments**.

Engineers completing this phase will be able to:

- understand how computers process and execute instructions
- operate confidently in Linux environments
- manage system processes and resources
- understand networking fundamentals
- write system-level scripts and tools
- control development environments using version control

This phase forms the **foundation upon which all future engineering skills are built**.

# 2. Core Learning Philosophy

The curriculum follows a **first-principles engineering approach**.

Engineers must understand the computing stack from the lowest levels upward.

```

Digital Logic
↓
Computer Architecture
↓
Operating Systems
↓
System Interaction
↓
Network Communication

```

Without this understanding, engineers struggle to build scalable and reliable systems.

Phase 1 ensures every engineer possesses **deep computational awareness**.

# 3. Module 1 — Binary Computation and Digital Logic

Computers operate using **binary representation and digital logic**.

Understanding these principles allows engineers to grasp how software instructions are translated into machine operations.

### Core Topics

- binary number system
- hexadecimal representation
- data encoding
- boolean logic
- logic gates
- digital circuits

### Key Concepts

**Binary Representation**

Computers store information using two states:

```

0  → Off
1  → On

```

All digital data is encoded using combinations of these bits.

Examples include:

- numbers
- characters
- instructions
- images

**Boolean Logic**

Logical operations form the foundation of computing.

Basic operations include:

- AND
- OR
- NOT
- XOR

These operations power decision-making in both hardware and software.

---

# 4. Module 2 — Computer Architecture

Computer architecture explains **how hardware components interact to execute programs**.

Engineers learn how instructions travel through the machine.

### Core Components

**Central Processing Unit (CPU)**

Responsible for executing instructions.

Key elements include:

- control unit
- arithmetic logic unit (ALU)
- registers

**Memory Systems**

Types of memory include:

- registers
- cache
- RAM
- persistent storage

Understanding memory hierarchy is critical for **performance optimization**.

**Instruction Cycle**

Programs execute through the following cycle:

```

Fetch Instruction
Decode Instruction
Execute Instruction
Store Result

```

Engineers learn how software instructions interact with hardware.

---

# 5. Module 3 — Operating Systems Fundamentals

Operating systems manage hardware resources and provide environments for running software.

Examples include:

- Linux
- Windows
- macOS

In engineering environments, **Linux is the dominant operating system**.

### Core Responsibilities of an Operating System

- process management
- memory management
- file systems
- hardware abstraction
- resource allocation

### Key Concepts

**Processes and Threads**

Programs run as processes.

Operating systems schedule and manage these processes to ensure efficient system performance.

**Memory Management**

Operating systems allocate memory dynamically to applications while preventing conflicts.

**File Systems**

Engineers must understand how files and directories are structured and accessed.

# 7. Module 5 — Networking Fundamentals

Modern software systems operate across distributed networks.

Engineers must understand how machines communicate.

### Core Networking Concepts

- network protocols
- IP addressing
- DNS resolution
- packet transmission
- HTTP communication

### Key Protocols

**TCP/IP**

Provides reliable communication between systems.

**HTTP**

Used for web communication and API requests.

**DNS**

Translates domain names into IP addresses.

Understanding these protocols allows engineers to build **network-enabled systems and services**.

# 8. Module 6 — Version Control and Engineering Workflow

Professional software development relies on **version control systems**.

### Core Concepts

- repositories

### Engineering Workflow

Typical workflow:

Clone repository

Commit changes
Push to repository
Open pull request
Merge changes

Version control enables:

- collaboration

- project history management
