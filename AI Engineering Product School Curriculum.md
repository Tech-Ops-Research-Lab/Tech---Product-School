# AI Engineering Product School Curriculum
### Ground-Up Engineering Program for AI Engineers

This document defines a **structured curriculum designed to produce AI Engineers from first principles**.  
The program trains individuals from **computational foundations to full AI system engineering and product deployment**.

The goal is to produce engineers capable of:

- understanding computer systems
- writing production-grade software
- building machine learning systems
- deploying AI infrastructure
- creating AI products
- documenting technical systems clearly

The curriculum progresses through **sequential engineering stages**, where each stage builds capabilities required for the next.

# 1. Curriculum Philosophy

AI engineers are not trained by teaching machine learning tools alone.

They are trained by understanding the **entire computational stack**:

```

Computer Systems
↓
Software Engineering
↓
Data Systems
↓
Machine Learning
↓
AI Infrastructure
↓
AI Product Development

```

Without the lower layers, engineers cannot build reliable AI systems.

The curriculum therefore moves **from computing fundamentals to advanced AI engineering**.

---

# 2. Engineering Capability Ladder

The program is structured into **ten sequential learning phases**.

```

Phase 1  → Computational Foundations
Phase 2  → Programming and Software Engineering
Phase 3  → Data and Mathematical Foundations
Phase 4  → Systems Engineering
Phase 5  → Data Engineering
Phase 6  → Machine Learning Engineering
Phase 7  → Deep Learning and AI Systems
Phase 8  → AI Infrastructure and Deployment
Phase 9  → AI Product Engineering
Phase 10 → Technical Writing and Documentation

```

Each stage builds capabilities required to operate at the next layer.

# 3. Phase 1 — Computational Foundations

Students begin by understanding **how computers function**.

## Core Topics

- binary computation
- computer architecture
- CPU and memory
- operating systems
- networking fundamentals
- Linux systems

## Key Skills

Students learn:

- how programs execute
- how operating systems manage processes
- how computers communicate through networks

## Tools

- Linux
- Bash
- Git
- Terminal environments

Outcome: engineers understand the **hardware–software relationship**.

- algorithms

Primary:

- JavaScript

## Data Structures

- stacks
- queues
- linked lists

# 13. Engineering Labs and Project-Based Learning

- architecture design
- data pipeline development
- model training

- full technical documentation
